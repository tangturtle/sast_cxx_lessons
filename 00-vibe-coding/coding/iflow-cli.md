# iFlow CLI

## 概览（Overview）

iFlow CLI 是一款“在终端中运行的 AI 助手/编码代理”，由阿里系/社区团队发起的免费 CLI 项目，主打即开即用、模型可切换（如 Qwen 系列、Kimi K2、DeepSeek 等）和典型开发工作流的自动化（读仓库 → 计划 → 生成补丁 → 运行/测试 → 迭代）。由于生态发展迅速，本文总结其在公开资料中常见的能力与用法，具体以官方仓库与文档为准。

## 核心能力（Key Features）

- 终端优先：在 Shell 中直接对话、生成/应用补丁（patch/diff）、查看说明。
- 仓库感知：读取当前 Git 仓库上下文（变更、依赖、测试目标）并据此给出修改方案。
- 多模型支持：可配置/切换多家 LLM 提供商与不同推理档位（以实际实现为准）。
- 工具调用：联动本地命令（lint、fmt、test、build），在失败时自动迭代修复。
- 任务化流程：将“修复/重构/实现/生成测试”等动作拆为可审阅的步骤与补丁。

## 适用场景（Use Cases）

- 本地仓库内的日常改动：小修小补、单测补齐、重构片段。
- 读日志/报错并提出修复：结合测试输出、构建日志、运行错误定位问题。
- 快速原型/脚手架：在现有模版基础上生成文件/模块初稿。

## 优势（Strengths）

- 贴合终端：无需离开 Shell，与脚本/CI 易集成。
- 可审计：以 patch/diff 为中心，更易做代码评审与回滚。
- 成本可控：可按需切换模型与推理参数，平衡质量/时延/价格。

## 局限（Limitations）

- 生态成熟度：与 Aider/Cline 等成熟 CLI 相比，社区与插件生态可能尚在建设。
- 权限与安全：应限制文件读写/命令执行范围，避免越权/误操作。
- 上下文边界：超大仓库或跨仓库依赖需要额外的索引/检索方案。

## 定价/获取（Pricing / Access）

- 开源/免费为主；以官方仓库与发行说明为准。
- 可结合自有/第三方 LLM API Key 使用，按 API 调用计费。

## 集成（Integration）

- Git 工作流：与分支/PR/提交信息生成配合。
- CI/CD：在 CI 中以“静默模式”运行生成补丁与报告（可参考 GitHub Action/脚本示例）。

## 对比（Comparison）

- vs Aider：两者均以补丁为核心；Aider 文档与社区较成熟，iFlow CLI 强调多模型与免费可用。
- vs Cline/Roo Code（CLI 模式）：后者偏“自治代理”流程，iFlow CLI 更贴近“我说你改”的回路。
- vs Gemini CLI/Claude Code：后者为厂商一方产品，平台集成度更高；iFlow CLI 更开放灵活。
