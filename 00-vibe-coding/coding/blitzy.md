# Blitzy

## 概览（Overview）

Blitzy 是一款面向企业软件开发的“自治型（autonomous）AI 平台”，主打“批量/票据驱动（batch/ticket-driven）”的端到端交付：在输入需求/规格后，由多智能体自动完成大部分（官方宣称 80%+）的代码生成、测试、重构、验证与提交，工程师聚焦于最后一公里的审查与定型。其卖点包括“无限代码上下文（infinite code context）”与在 SWE-bench 等评测中的表现声明。

注：厂商信息更新较频繁，下述为对公开资料中稳定能力的概括；具体订阅、形态与可用区域以官网为准。

## 核心能力（Key Features）

- 批量/票据驱动：围绕需求/任务，将工作拆分为可审阅的子步骤与补丁。
- 多智能体编排：规划/实现/测试/审阅/合规等角色协同推进。
- 上下文与知识：强调大规模代码上下文与跨仓理解（以官方实现为准）。
- 质量门禁：单测/集成测试/静态分析/风控与回滚策略（以企业版为准）。
- 可观测与治理：动作轨迹、输入输出、补丁与日志可追踪；便于审计复盘。

## 典型工作流（Workflow）

1. 输入规格/需求或选择现有票据；
2. 平台生成任务分解与执行计划；
3. 产出 patch/diff 与说明，必要时请求人审批；
4. 运行测试与质量门禁，失败则回退/重试；
5. 生成 PR 并附带变更摘要与验证步骤。

## 优势（Strengths）

- 端到端闭环：从需求到 PR 的自治化串联，缩短交付周期。
- 工程治理：以补丁/门禁/审计为边界，便于合规与复盘。
- 大上下文：强调大仓/跨仓理解与上下文构建（以官方为准）。

## 局限（Limitations）

- 黑盒程度：模型与路由策略细节不完全公开，需试用评估；
- 依赖生态：对测试覆盖率、CI 质量门禁与权限治理有依赖；
- 需求澄清：在含糊需求与领域知识缺乏时仍需人工澄清与验收。
