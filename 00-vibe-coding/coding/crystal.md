# Crystal

## 概览（Overview）

Crystal 是一款面向“多会话、多上下文”的 AI 编程工作台（常与 Claude Code/Codex 协同），用于并行推动多个任务或分支，管理上下文、差异与补丁，减少不同任务之间的相互污染与遗忘。

注：社区资料显示其强调“会话隔离、补丁导出、与 IDE/CLI 协作”。以下为通用能力梳理，具体以官方为准。

## 核心能力（Key Features）

- 多会话并行：针对不同问题开独立会话，互不干扰。
- 上下文管理：为每个会话维护独立的文件/指令/历史。
- 补丁/差异视图：以 diff/patch 形式导出修改，便于审阅与合并。
- 同步 CLI/IDE：与本地编辑器/终端协作，支持运行/测试/回滚。
- 模型可配置：按任务选择不同模型与推理参数（以官方实现为准）。

## 适用场景（Use Cases）

- 并行推进多条开发线（bug 修复、功能开发、文档整理等）。
- 大型重构/迁移，按子域开会话、逐步合并。
- 教学/演示：同一仓库内保留多个解法路径。

## 优势（Strengths）

- 降低“上下文串味”：不同任务互不干扰。
- 更易代码审查：补丁化输出便于 PR 审阅。
- 贴近工程实践：与 Git/CI 工作流契合。

## 局限（Limitations）

- 需要建立“会话即工作单元”的习惯。
- 与 IDE/CI 的集成深度决定落地体验。

## 定价/获取（Pricing / Access）

- 以官网与仓库为准。

## 对比（Comparison）

- vs Cursor/Windsurf：后者更像 IDE；Crystal 是“多会话/补丁中枢”。
- vs Cline/Roo Code：它们偏“自治代理”；Crystal 偏“会话/变更管理”。
