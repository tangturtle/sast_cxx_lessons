# OnSpace.AI

## 概览（Overview）

OnSpace.AI 是一个面向业务团队与个人的 AI 原生「无/低代码」应用构建平台，主打通过“上下文工程（Context Engineering）”方法，将数据源、知识与流程结构化，快速生成可运行的 Web/工作流应用。其定位介于可视化应用搭建器与 AI 代理/自动化平台之间。

注：本文总结公开资料中的能力与架构要点；具体产品功能与价格以官方为准。

## 核心能力（Key Features）

- 上下文工程架构：对数据、指令、约束进行结构化建模，提高生成的稳定性与可控性。
- 数据/知识接入：支持接入文档、知识库、API、表格等数据源，统一纳入上下文。
- 可视化编排：通过可视化界面定义页面、表单、工作流与触发器，无需大量手写代码。
- 代理与自动化：将 LLM 与工具链封装为可复用的步骤，串联为端到端业务流程。
- 权限与发布：提供用户/角色管理、应用发布与运行监控（以官方实现为准）。

## 适用场景（Use Cases）

- 业务原型/内部工具：在数小时到数天内完成从需求到可用应用的闭环。
- 数据助理/知识问答：面向企业知识库构建问答与流程自动化助手。
- 表单/工作流应用：审批、收集、路由、通知等典型流程应用。

## 优势（Strengths）

- 更强的“工程化”生成：上下文结构化降低“幻觉”和不稳定输出。
- 上手速度快：无需搭建代码框架/后端，适合小团队快速试验。
- 与 AI/自动化结合紧密：相比传统低代码更好地承载 LLM 能力。

## 局限（Limitations）

- 复杂后端/专用基础设施：在高复杂度/高并发/强合规场景仍可能需要工程团队介入。
- 上下文与数据治理：需要合理的权限边界、数据分层与可观测性策略。
- 供应商耦合：可移植性取决于导出/API 能力与生态配套。

## 对比（Comparison）

- vs Bolt.new / v0.dev：后者偏“从文本到前端/代码”，OnSpace 更强调上下文工程与流程。
- vs Replit / Lovable：后者偏在线 IDE/代码生成；OnSpace 偏可视化编排与业务应用。
- vs 传统低代码：OnSpace 与 LLM/代理结合更紧，但在底层可控性上需结合导出/扩展能力评估。
