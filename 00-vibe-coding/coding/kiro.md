# Kiro

## 概览（Overview）

Kiro 是一款“规格驱动（spec‑driven）”的 AI‑优先开发工具（常被称为 AI IDE），强调用结构化需求与验收标准来驱动代理分解、实现与回归。官方资料显示其面向从原型到生产的全链路过程，聚焦“把 Prompt 变成清晰需求与任务”的工程化落地。

注：生态快速演进，以下为通用能力总结；具体功能/订阅与可用地区以官方为准。

## 核心能力（Key Features）

- 规格驱动开发（Spec‑driven）：将需求转为结构化规格、任务与里程碑。
- 多代理规划与执行：按角色（分析/实现/测试/回顾）协同推进。
- 端到端闭环：从设计 → 代码 → 测试/验收 → 维护的循环。
- 长上下文与项目记忆：在同一项目中复用约定、架构与代码风格。
- 与现有工具协作：Git 分支/PR 工作流、问题追踪与评审流程（以官方为准）。

## 适用场景（Use Cases）

- 新功能从需求到交付的端到端推进。
- 复杂重构/升级，需要规划、回归与质量门禁。
- 团队“反‑vibe coding”：用明确规格替代随意对话式开发。

## 优势（Strengths）

- 规格化与可衡量：把 Prompt 抽象为可验收的工程制品。
- 团队流程契合：更容易与 PR/评审/回归测试结合。
- 复杂任务可控：多代理分工减少“长对话走偏”。

## 局限（Limitations）

- 初期上手成本：需要转变为“写规格/验收准则”的工作方式。
- 与现有堆栈的集成深度需评估（CI/安全/私有依赖等）。

## 对比（Comparison）

- vs Cursor/Windsurf/Zed：后者偏 IDE + Copilot；Kiro更强调“规格驱动/多代理规划 + 端到端”。
- vs Devin/企业平台：定位接近，但实现细节、可部署形态、可观测性不同。
