# Goose

概览 / Overview

- 定位：Block 开源的可扩展 AI 编程代理（CLI + Desktop），在终端中帮助安装、执行、编辑、测试，超越“仅建议”的补全形态。
- 典型场景：从需求到实现的闭环自动化（创建项目、修改代码、运行脚本、执行测试、收集日志、继续迭代）。
- 适用人群：偏工程化/自动化的开发者、DevOps、愿意让代理“动手做事”的团队。

Key Features（核心特性）

- 终端代理：通过命令行与 Goose 交互，允许其在获批后执行命令、编辑文件、运行测试。
- 多模型支持：可接入多种 LLM（Claude、Gemini、GPT、开源模型等），支持自定义提供者与密钥管理。
- 可扩展工具链：插件化/工具化能力，按需增加“安装依赖、运行构建、集成测试”等步骤。
- 上下文管理：基于项目目录与会话历史驱动的上下文，支持持续迭代与回溯。
- 桌面版本：Goose Desktop 进一步提供可视化与更丰富的人机交互体验（与 CLI 共享理念）。

Strengths（优势）

- 工程闭环：不仅“说”，还能“做”——安装、执行、编辑与测试贯穿始终。
- 开源与可塑：公开代码与扩展接口，便于企业/团队按需自定义。
- 模型无关：根据成本/质量/时延切换不同模型或走自托管路线。

Limitations（限制）

- 审批成本：安全考虑下的每步确认可能增加交互成本，需要配置合理的权限与策略。
- 学习与治理：代理可执行命令，需配合最小权限、沙箱/容器或非生产环境实践。
- 生态成熟度：与传统 IDE 插件生态相比，插件/脚本生态仍在快速演进中。

Use Cases（典型用例）

- 从需求到补丁：基于 issue/描述生成补丁，并自动运行测试验证。
- 快速原型：初始化项目、安装依赖、生成骨架代码并启动开发服务器。
- 运维脚本：自动化执行常见维护任务（备份、日志清理、服务滚动重启等）。

Pricing / Access（获取与定价）

- 开源（block/goose）：遵循仓库许可；模型费用由所选提供者计费。
- 可在本地或企业环境中部署与使用，自定义网络/代理/缓存等策略。

Integrations（集成）

- 终端/壳：bash、zsh、PowerShell
- 工具：Git、包管理器、测试框架、容器/脚本执行器（取决于环境）
- 模型提供者：OpenAI、Anthropic、Google、开源推理后端等

Comparison（对比要点）

- 对比 Gemini CLI：Goose 更偏开源多模型与可塑性；Gemini CLI 更贴近谷歌生态与官方工具。
- 对比 Aider：Aider 强于 Git 补丁/最小变更与审阅；Goose 更强调“执行命令+自动化流水线”。
- 对比 OpenCode：两者都主打终端代理；OpenCode 拥有原生 TUI 与 SST 生态，Goose 则更强调可扩展与多模型。
